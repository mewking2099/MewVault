# MewVault — what changed on 2026-07-08

One-day overhaul: token audit, health system, agent visibility, wiki retrieval loop, design silo v2, software silo spec-driven pipeline, and natural-language command routing. This doc is the reference for sessions and humans alike.

## 1. Token burn fixed (root cause: Headroom)

The Headroom compression proxy rewrote the prompt prefix every request, breaking Anthropic prompt caching (cache reads ≈ 0.1x input cost) — compression was a large net loss on subscription. Its compress/retrieve tools also duplicated content in the append-only context window.

- Headroom disabled: launchd daemon removed, `ANTHROPIC_BASE_URL` cleared, `start-headroom.sh` refuses without `MEW_HEADROOM_FORCE=1`, MCP config marked disabled. Postmortem: `wiki/headroom-postmortem.md`.
- Rule going forward: never put a request-rewriting proxy between Claude Code and the API. Optimize by injecting less, never by transforming the prompt.
- Session-start truncation now drops whole low-priority sections instead of substring cuts; trigger instructions and the Session Card are never truncated.
- Hook matchers fixed: PreToolUse/PostToolUse scoped (`Bash|Write|Edit|MultiEdit`, `Write|Edit|MultiEdit`); `mew harness install` no longer erases matchers.
- Webflow MCP scoped to golddiamondclub only (was global in every session).
- Instinct pipeline tamed: 3+ rapid rewrites required (was 2), deduped per topic, queue capped at 50, stale signals pruned; 942 junk candidates archived.

## 2. Health system — `mew doctor`

15 automated checks: cache_safety, cache_burn (cache-hit ratio from transcripts), hooks_registered, hook_matchers, injection_size (runs the hook and measures), impeccable, ci_presence, wip_limit, instinct_queue, signal_file, mcp_surface, ollama, index_freshness. Runs detached on the first prompt of every session; macOS notification on warn/fail; issues injected as a `## Vault Health` section next session. Manual: `mew doctor`, `--json`, `--quiet`, `--notify`.

## 3. Agent visibility + enforcement

- `hooks/agent-track.js` (PreToolUse matcher `Task` + `SubagentStop`): every agent dispatch logged to `.claude/agent-dispatches.jsonl`; mew-* dispatches **without a model param are blocked** (the silent-Sonnet-fallback fix).
- `mew agent status` — dispatches, completions, blocked count, model actually used.

## 4. Dashboard + usage

- `mew dashboard` — self-contained HTML at `.claude/dashboard.html`: all projects (phase, next action, idle days, audit P0), doctor health, agent activity. `--watch N` regenerates in a loop.
- `mew usage --report [--days N]` — per-day tokens + cache-hit ratio from Claude Code transcripts.

## 5. MewWiki retrieval loop closed

- `mew wiki sync` auto-runs the doobidoo re-index (background, skips if Ollama down; `MEW_AUTO_INGEST=off` to disable).
- Session start instructs Claude to consult `mcp__doobidoo__retrieve_memory` before substantive work — past decisions and gotchas resurface instead of rotting.

## 6. Design silo v2 (the deep one)

- **Impeccable v3 enforced, all silos**: post-tool-use guard on every UI file write — absolute-ban detector (side-stripes, gradient text, glassmorphism, radius > 16px, rounded-3xl) + a once-per-session flow anchor when UI work starts. Rules updated to the four-phase loop (init → iterate → audit/clarify/harden → extract/document). Anthropic's colliding `frontend-design` skill disabled (`skills/_disabled/`).
- **Audit-score gate**: `/impeccable audit` results persist to Project_Status.md (`last_audit`, `audit_scores`, `open_p0`); moving a design project to handoff/delivery with `open_p0 > 0` is hook-blocked.
- `mew design tokens --diff` — Figma variables vs DESIGN.md drift (two-step: Claude saves `assets/figma-variables.json` via the Figma MCP, CLI diffs).
- `mew package <project> --design` — full handoff: PRODUCT.md, DESIGN.md, audit summary, decision log from mewwiki, assets manifest.
- `scripts/snapshot.mjs` — visual regression on wrap (needs `snapshot.routes.json` in project root + Playwright).
- Standup now reports unresolved Figma comments per design project; design `decision` dumps require Figma frame provenance.

## 7. Software silo — spec-driven development

- **`spec <feature>` trigger**: raw brief → `specs/<feature>.md` (numbered AC-n acceptance criteria, Given/When/Then) → hard stop for owner approval → only then tests and code. Template: `templates/spec.md.tmpl`.
- **TDD gate is now a hard block** on stalk/mewking projects (source file with no test = blocked; opt out per project with `tdd: off`). Pounce warns only.
- **Definition of Done at wrap**: code projects run typecheck/lint/test/build before the log entry; failures tag it `[incomplete]` — no "done" over a red check.
- **CI safety net**: `mew ci install` — workflow (typecheck, lint, test, build, npm audit) installed into dsaas, mew-knowledge-hub, mewvault-site, yaana-design-system; baked into new-project scaffolds (plus `specs/`, `tests/`, `raw/` dirs).
- Doctor enforces WIP: warns at >3 active projects or any active project idle 21+ days.

## 8. Natural-language command routing

You never need exact commands. Say it in plain English; the session-start trigger router runs the right thing:

| Say | Runs |
|---|---|
| "doctor" / "health check" / "is everything ok" | mew doctor |
| "dashboard" / "vault overview" | mew dashboard |
| "agent status" / "did agents run" | mew agent status |
| "token report" / "burn rate" / "cache hit" | mew usage --report |
| "install ci" | mew ci install |
| "token drift" / "check tokens" | mew design tokens --diff |
| "prepare handoff <project>" | mew package <project> --design |
| "sync wiki" | mew wiki sync |
| "spec <feature>" | spec-driven workflow |
| "critique <url>" | pixel critique workflow |
| "design session <project>" | Impeccable session setup |
| "weekly review" | week digest → Brain/Memories |
| standup · wrap up · dump · meeting prep | (as before) |
